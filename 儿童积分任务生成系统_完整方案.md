## 一、系统概述

### 1.1 系统名称

**童趣积分乐园**

### 1.2 目标用户

- **儿童**：4-12岁，通过完成任务获取积分
- **家长**：管理孩子的学习计划和积分

### 1.3 核心价值

通过游戏化激励机制，激发儿童学习兴趣，培养良好习惯

### 1.4 积分体系

| 积分类型 | 获取方式 | 用途 |
|----------|----------|------|
| **普通积分** | 每日打卡、完成基础任务 | 兑换普通礼品 |
| **奖励积分** | 及时完成作业、作品优秀、完成额外任务 | 兑换高级礼品 |

## 二、功能模块设计

### 2.1 用户角色与权限

| 角色 | 核心权限 |
|------|----------|
| **儿童** | 打卡任务、上传作品、兑换礼品、查看积分 |
| **家长** | 生成任务、审核打卡、调整积分、管理孩子、查看报告 |

### 2.2 核心功能模块

#### 模块一：用户管理
- 统一登录页面（自动判断角色）
- 首次进入收集儿童信息（姓名、年龄、爱好、年级）
- 生成测试题评估基础知识水平
- 生成人物简报

#### 模块二：任务系统
- AI一键生成任务
- 自定义任务模板（337阅读法、计算题练习等）
- 教材管理（添加教材、设置进度）
- 任务状态管理（开启/暂停）

#### 模块三：打卡系统
- 支持图片/视频/文字打卡
- AI自动批改作业
- 生成巩固练习题
- 家长审核流程

#### 模块四：积分商城
- 礼品分类展示
- 积分兑换流程
- 家长确认发货

#### 模块五：学习报告
- 数据可视化仪表盘
- 学习趋势分析
- 能力评估报告
- 目标设定与时间管理

## 三、前端设计方案

### 3.1 技术选型

| 分类 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | React | 18.x | 主流框架，生态成熟 |
| UI框架 | Ant Design | 5.x | 企业级组件库 |
| 图标库 | Ant Design Icons + Lucide React | - | 图标资源 |
| 状态管理 | Zustand | 4.x | 轻量状态管理 |
| 路由 | React Router DOM | 6.x | 单页应用路由 |
| HTTP请求 | Axios | 1.x | API请求封装 |
| 图表库 | ECharts React | 3.x | 数据可视化 |
| 构建工具 | Vite | 6.x | 快速构建 |
| 样式 | TailwindCSS | 3.x | 原子化CSS |

### 3.2 页面结构

#### 统一登录页
- 路径：`/login`
- 功能：统一登录入口，自动判断角色

#### 儿童端页面

| 页面 | 路径 | 功能 |
|------|------|------|
| 信息收集 | `/child/onboarding` | 首次信息收集 |
| 首页 | `/child/home` | 任务列表、积分展示 |
| 任务详情 | `/child/task/:id` | 任务详情、打卡提交 |
| 积分商城 | `/child/mall` | 礼品浏览、兑换 |
| 个人中心 | `/child/profile` | 个人信息、积分记录 |

#### 家长端页面

| 页面 | 路径 | 功能 |
|------|------|------|
| 仪表盘 | `/parent/dashboard` | 数据概览 |
| 任务管理 | `/parent/tasks` | 任务列表、AI生成 |
| 打卡审核 | `/parent/checkins` | 打卡审核 |
| 教材管理 | `/parent/textbooks` | 教材管理 |
| 学习报告 | `/parent/reports` | 数据分析报告 |
| 孩子管理 | `/parent/children` | 孩子信息、目标设定 |

### 3.3 路由配置逻辑

```
登录页 → 判断角色 → 儿童端 / 家长端
         ↓
    角色判断逻辑：
    - 检查用户信息中的 role 字段
    - role === 'child' → 跳转 /child/home
    - role === 'parent' → 跳转 /parent/dashboard
```

## 六、部署方案

### 6.1 环境要求

| 环境 | 要求 |
|------|------|
| Node.js | >= 18.0.0 |
| npm | >= 9.0.0 |

### 6.2 安装步骤

```bash
# 1. 克隆项目
git clone <repository-url>
cd childer_manager

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置飞书API等参数

# 4. 开发模式
npm run dev

# 5. 生产构建
npm run build

# 6. 部署
# 将 dist 目录部署到静态服务器
```

### 6.3 环境变量配置

```env
# 飞书多维表格配置
REACT_APP_LARK_APP_TOKEN=your-app-token
REACT_APP_LARK_TABLE_USER=users-table-id
REACT_APP_LARK_TABLE_TASK=tasks-table-id
REACT_APP_LARK_TABLE_CHECKIN=checkins-table-id
REACT_APP_LARK_TABLE_GIFT=gifts-table-id
REACT_APP_LARK_TABLE_TEXTBOOK=textbooks-table-id

# API配置
REACT_APP_API_BASE_URL=http://localhost:3000/api

# 开发配置
REACT_APP_DEBUG=true
```

## 七、安全性设计

1. **登录认证**：飞书 OAuth 2.0 登录
2. **数据加密**：HTTPS 传输
3. **权限控制**：基于角色的访问控制（RBAC）
4. **内容审核**：AI 自动审核上传内容
5. **XSS 防护**：使用 React 自动转义
6. **数据备份**：飞书多维表格自动备份
